    search_crew Tool¶

    Overview¶

    The search_crew tool enables LLMs to search for crew members using natural language queries combined with structured filters. This tool provides AI-powered matching capabilities that understand maritime terminology and crew management context.

    Tool Signature¶

    async def search_crew(
    query: str,
    filters: Optional[Dict[str, Any]] = None,
    limit: int = 10,
    db: AsyncSession = None
) -> Dict[str, Any]

    Parameters¶

    Required Parameters¶

    Parameter Type Description Example
    query string Natural language search query "experienced deck officers with valid STCW"

    Optional Parameters¶

    Parameter Type Default Description Example
    filters object null Additional structured filters {"department": "deck", "status": "active"}
    limit integer 10 Maximum number of results (1-100) 25

    Filter Options¶

    The filters object supports the following properties:

    Filter Type Description Valid Values
    department string Filter by crew department "deck", "engine", "catering", "other"
    rank string Filter by specific rank/position "Captain", "Chief Officer", "Engineer", etc.
    status string Filter by employment status "active", "inactive", "on_leave"
    vessel_id string Filter by current vessel assignment Valid vessel UUID
    nationality string Filter by crew member nationality ISO country codes

    Response Schema¶

    Successful Response¶

    {
  "query": "string",           // Original search query
  "filters": "object|null",    // Applied filters
  "matches": [                 // Array of matching crew members
    {
      "id": "string",          // Crew member UUID
      "name": "string",        // Full name
      "rank": "string|null",   // Current rank/position
      "department": "string|null", // Department
      "status": "string|null", // Employment status
      "email": "string|null",  // Contact email
      "nationality": "string|null", // Nationality
      "relevance_score": "number"  // Relevance score (0.0-1.0)
    }
  ],
  "total_count": "integer",    // Number of matches returned
  "limit": "integer",          // Applied limit
  "search_time": "number"      // Search execution time (seconds)
}

    Error Response¶

    {
  "error": "string",           // Error description
  "matches": [],               // Empty array
  "total_count": 0,           // Zero count
  "query": "string|null",     // Original query if available
  "filters": "object|null"    // Applied filters if available
}

    Natural Language Understanding¶

    The search tool understands various maritime terms and contexts:

    Rank Recognition¶

    • "officers" → Searches for officer-level positions
    • "deck officers" → Filters to deck department officers
    • "engineers" → Searches for engineering positions
    • "captain", "chief" → Searches for senior positions

    Department Context¶

    • "deck crew" → Filters to deck department
    • "engine room" → Filters to engine department
    • "galley", "catering" → Filters to catering department

    Experience Indicators¶

    • "experienced" → Prioritizes crew with longer service
    • "senior" → Focuses on higher ranks
    • "qualified" → Emphasizes certification status

    Certification Context¶

    • "valid certificates" → Considers current certification status
    • "STCW" → Focuses on STCW-certified crew
    • "DPO" → Searches for Dynamic Positioning Operators

    LLM Integration Examples¶

    Basic Search Implementation¶

    # Example LLM prompt processing
def process_crew_search_request(user_query: str):
    """
    Process user's natural language request into search_crew tool call
    """

    # Extract intent and parameters from user query
    query, filters = parse_search_intent(user_query)

    # Execute search
    result = await search_crew(
        query=query,
        filters=filters,
        limit=10
    )

    # Format response for user
    if result.get("matches"):
        return format_crew_search_results(result)
    else:
        return "No crew members found matching your criteria."

def format_crew_search_results(result: dict) -> str:
    """Format search results for natural language response"""
    matches = result["matches"]
    total = result["total_count"]

    response = f"Found {total} crew member(s):\n\n"

    for crew in matches:
        response += f"• {crew['name']}"
        if crew['rank']:
            response += f" - {crew['rank']}"
        if crew['department']:
            response += f" ({crew['department']} dept.)"
        response += "\n"

    return response

    Advanced Search with Context¶

    class CrewSearchAssistant:
    """AI assistant for crew search operations"""

    async def handle_search_query(self, query: str, context: dict = None):
        """Handle complex search queries with context awareness"""

        # Parse query for implicit filters
        parsed = self.parse_search_context(query, context)

        # Execute search with parsed parameters
        result = await search_crew(
            query=parsed["query"],
            filters=parsed["filters"],
            limit=parsed.get("limit", 10)
        )

        # Provide contextual response
        return self.generate_contextual_response(result, query, context)

    def parse_search_context(self, query: str, context: dict) -> dict:
        """Parse search query with contextual information"""

        filters = {}

        # Apply contextual filters
        if context.get("current_vessel"):
            filters["vessel_id"] = context["current_vessel"]["id"]

        if context.get("target_department"):
            filters["department"] = context["target_department"]

        # Extract implicit filters from query
        if "available" in query.lower():
            filters["status"] = "active"

        if "deck" in query.lower():
            filters["department"] = "deck"
        elif "engine" in query.lower():
            filters["department"] = "engine"

        return {
            "query": query,
            "filters": filters,
            "limit": self.extract_limit(query)
        }

    Performance Considerations¶

    Search Optimization¶

    1. Index Usage: The tool leverages database indexes on commonly searched fields
    2. Query Complexity: Complex natural language queries may take longer to process
    3. Result Limits: Use appropriate limits to balance completeness and performance
    4. Filter Efficiency: Structured filters are more efficient than text search

    Recommended Practices¶

    # Good: Specific filters with reasonable limits
await search_crew(
    query="experienced officers",
    filters={"department": "deck", "status": "active"},
    limit=20
)

# Less efficient: Very broad search with high limits
await search_crew(
    query="crew",
    limit=100
)

# Good: Combine text search with filters
await search_crew(
    query="STCW certified",
    filters={"status": "active"},
    limit=15
)

    Error Handling in LLM Integration¶

    async def safe_crew_search(query: str, **kwargs):
    """Wrapper for safe crew search with error handling"""

    try:
        result = await search_crew(query=query, **kwargs)

        if "error" in result:
            # Handle tool-specific errors
            return {
                "success": False,
                "message": f"Search failed: {result['error']}",
                "suggestions": get_search_suggestions(query)
            }

        return {
            "success": True,
            "data": result
        }

    except Exception as e:
        # Handle unexpected errors
        logger.error(f"Crew search error: {str(e)}")
        return {
            "success": False,
            "message": "An unexpected error occurred during search.",
            "suggestions": ["Try simplifying your search query", "Check your filters"]
        }
