    Appraisals API¶

    The Appraisals API provides comprehensive endpoints for managing crew performance evaluations, including creating appraisals, tracking performance metrics, managing approval workflows, and generating reports.

    Overview¶

    The Performance Appraisal system enables systematic evaluation of crew members' performance, skills, and competencies. It supports:

    • Performance Evaluations: Structured assessment of crew member performance
    • Skills Assessment: Evaluation of technical and soft skills
    • Goal Setting: Setting and tracking performance objectives
    • Approval Workflows: Multi-level approval process for appraisals
    • Historical Tracking: Performance trends and improvement tracking
    • Reporting: Performance analytics and insights

    Appraisal Process¶

    Workflow Stages¶

    stateDiagram-v2
    [*] --> Draft
    Draft --> In_Review: Submit for Review
    In_Review --> Approved: Supervisor Approves
    In_Review --> Needs_Revision: Request Changes
    Needs_Revision --> In_Review: Resubmit
    Approved --> Final: HR Final Approval
    In_Review --> Rejected: Reject
    Final --> [*]

    Appraisal Types¶

    Type Description Frequency Participants
    annual Annual performance review Yearly Supervisor, HR, Self
    probation Probationary period review After probation Supervisor, HR
    promotion Promotion assessment As needed Senior Management, HR
    departure Exit evaluation Upon leaving Supervisor, HR
    360_feedback Multi-source feedback Semi-annually Peers, Subordinates, Supervisors

    Data Model¶

    Appraisal Object¶

    Field Type Description
    id UUID Unique appraisal identifier
    crew_member_id UUID Crew member being appraised
    appraiser_id UUID Primary appraiser (supervisor)
    period_start date Appraisal period start date
    period_end date Appraisal period end date
    type string Type of appraisal
    status string Current appraisal status
    overall_rating number Overall performance rating (1-5)
    vessel_id UUID Vessel where performance was observed

    Performance Categories¶

    The system evaluates performance across multiple categories:

    Category Description Weight
    Technical Skills Job-specific technical competencies 30%
    Safety Performance Safety awareness and compliance 25%
    Communication Communication effectiveness 15%
    Leadership Leadership and teamwork abilities 15%
    Reliability Dependability and work ethic 15%

    Rating Scale¶

    Rating Description Criteria
    5 Exceptional Consistently exceeds expectations
    4 Above Average Frequently exceeds expectations
    3 Meets Expectations Consistently meets job requirements
    2 Below Average Sometimes meets expectations
    1 Unsatisfactory Does not meet basic requirements

    Appraisal Status¶

    Status Description
    draft Initial creation, not yet submitted
    in_review Submitted for supervisor review
    needs_revision Returned for modifications
    approved Approved by supervisor
    final Final approval by HR
    rejected Rejected and closed

    Permissions¶

    Permission Description Allowed Roles
    appraisal.read View appraisals All authenticated users
    appraisal.write Create/edit appraisals supervisor, hr_manager, admin
    appraisal.approve Approve appraisals supervisor, hr_manager, admin
    appraisal.reports Access appraisal reports hr_manager, admin

    Role-Based Access¶

    • Crew Members: Can view their own appraisals only
    • Supervisors: Can create/edit appraisals for their team members
    • HR Managers: Full access to all appraisals and reports
    • Administrators: Complete system access

    Key Features¶

    Self-Assessment¶

    Crew members can complete self-assessments as part of the appraisal process:

    • Rate their own performance in each category
    • Set personal development goals
    • Provide comments on achievements and challenges
    • Request specific training or development opportunities

    360-Degree Feedback¶

    For comprehensive evaluation:

    • Collect feedback from multiple sources
    • Peer evaluations for teamwork assessment
    • Subordinate feedback for leadership roles
    • Customer/client feedback where applicable

    Goal Setting and Tracking¶

    • Set SMART goals for the next period
    • Link goals to organizational objectives
    • Track progress throughout the period
    • Measure achievement at appraisal time

    Development Planning¶

    • Identify skill gaps and improvement areas
    • Create individual development plans
    • Recommend training programs
    • Set career progression paths

    Endpoints¶

    Core Appraisal Operations¶

    • GET /appraisals - List appraisals with filtering
    • POST /appraisals - Create new appraisal
    • GET /appraisals/{id} - Get appraisal details
    • PUT /appraisals/{id} - Update appraisal
    • DELETE /appraisals/{id} - Delete appraisal

    Workflow Operations¶

    • POST /appraisals/{id}/submit - Submit for review
    • POST /appraisals/{id}/approve - Approve appraisal
    • POST /appraisals/{id}/reject - Reject appraisal
    • POST /appraisals/{id}/request-revision - Request changes

    Assessment Operations¶

    • POST /appraisals/{id}/self-assessment - Submit self-assessment
    • GET /appraisals/{id}/feedback - Get 360 feedback
    • POST /appraisals/{id}/feedback - Add feedback

    Reporting and Analytics¶

    • GET /appraisals/reports/summary - Appraisal summary report
    • GET /appraisals/reports/performance-trends - Performance trends
    • GET /appraisals/reports/department-stats - Department statistics
    • GET /appraisals/crew/{id}/history - Crew member appraisal history

    Integration Examples¶

    Basic Appraisal Management¶

    import requests
from datetime import date, timedelta

class AppraisalAPI:
    def __init__(self, base_url: str, access_token: str):
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {access_token}",
            "Content-Type": "application/json"
        }

    def create_appraisal(self, crew_member_id: str, appraisal_data: dict):
        """Create a new appraisal"""
        response = requests.post(
            f"{self.base_url}/api/v1/appraisals",
            json={
                "crew_member_id": crew_member_id,
                **appraisal_data
            },
            headers=self.headers
        )
        return response.json() if response.status_code == 201 else None

    def list_pending_appraisals(self):
        """List appraisals requiring review"""
        response = requests.get(
            f"{self.base_url}/api/v1/appraisals",
            params={"status": "in_review"},
            headers=self.headers
        )
        return response.json() if response.status_code == 200 else None

    def approve_appraisal(self, appraisal_id: str, comments: str = None):
        """Approve an appraisal"""
        data = {"approved": True}
        if comments:
            data["comments"] = comments

        response = requests.post(
            f"{self.base_url}/api/v1/appraisals/{appraisal_id}/approve",
            json=data,
            headers=self.headers
        )
        return response.status_code == 200

    def get_crew_performance_history(self, crew_member_id: str):
        """Get performance history for crew member"""
        response = requests.get(
            f"{self.base_url}/api/v1/appraisals/crew/{crew_member_id}/history",
            headers=self.headers
        )
        return response.json() if response.status_code == 200 else None

api = AppraisalAPI("http://localhost:8000", access_token)

# Create annual appraisal

    "type": "annual",
    "period_start": str(date.today() - timedelta(days=365)),
    "period_end": str(date.today()),
    "vessel_id": "vessel-uuid-here"
}

appraisal = api.create_appraisal("crew-member-uuid", appraisal_data)
if appraisal:
    print(f"Created appraisal: {appraisal['id']}")

    # Add performance ratings
    ratings = {
        "technical_skills": 4,
        "safety_performance": 5,
        "communication": 3,
        "leadership": 4,
        "reliability": 4
    }
    # Update with ratings...

# List pending reviews

if pending:
    for appraisal in pending['items']:

if history:

    class AppraisalManager {
    constructor(baseUrl, accessToken) {
        this.baseUrl = baseUrl;
        this.headers = {
            'Authorization': `Bearer ${accessToken}`,
            'Content-Type': 'application/json'
        };
    }

    async createAppraisal(crewMemberId, appraisalData) {
        const response = await fetch(`${this.baseUrl}/api/v1/appraisals`, {
            method: 'POST',
            headers: this.headers,
            body: JSON.stringify({
                crew_member_id: crewMemberId,
                ...appraisalData
            })
        });
        return response.ok ? await response.json() : null;
    }

    async submitForReview(appraisalId, assessmentData) {
        const response = await fetch(
            `${this.baseUrl}/api/v1/appraisals/${appraisalId}/submit`,
            {
                method: 'POST',
                headers: this.headers,
                body: JSON.stringify(assessmentData)
            }
        );
        return response.ok;
    }

    async listMyAppraisals() {
        const response = await fetch(`${this.baseUrl}/api/v1/appraisals`, {
            headers: this.headers
        });
        return response.ok ? await response.json() : null;
    }

    async getAppraisalDetails(appraisalId) {
        const response = await fetch(
            `${this.baseUrl}/api/v1/appraisals/${appraisalId}`,
            { headers: this.headers }
        );
        return response.ok ? await response.json() : null;
    }

    async provideFeedback(appraisalId, feedback) {
        const response = await fetch(
            `${this.baseUrl}/api/v1/appraisals/${appraisalId}/feedback`,
            {
                method: 'POST',
                headers: this.headers,
                body: JSON.stringify(feedback)
            }
        );
        return response.ok;
    }
}

// Usage
const appraisalManager = new AppraisalManager('http://localhost:8000', accessToken);

// Create and submit self-assessment
async function completeSelfAssessment(crewMemberId) {
    const appraisal = await appraisalManager.createAppraisal(crewMemberId, {
        type: 'annual',
        period_start: '2023-01-01',
        period_end: '2023-12-31'
    });

    if (appraisal) {
        const selfAssessment = {
            technical_skills: 4,
            safety_performance: 5,
            communication: 3,
            goals_achieved: [
                "Completed advanced navigation course",
                "Improved bridge team coordination"
            ],
            development_areas: [
                "Leadership skills",
                "Advanced cargo operations"
            ],
            comments: "Had a successful year with good safety record"
        };

        await appraisalManager.submitForReview(appraisal.id, selfAssessment);
        console.log('Self-assessment submitted for review');
    }
}

// Review pending appraisals (supervisor view)
async function reviewPendingAppraisals() {
    const appraisals = await appraisalManager.listMyAppraisals();

    if (appraisals) {
        const pending = appraisals.items.filter(a => a.status === 'in_review');

        for (const appraisal of pending) {
            console.log(`Review needed: ${appraisal.crew_member_name}`);

            // Get detailed appraisal data
            const details = await appraisalManager.getAppraisalDetails(appraisal.id);
            if (details) {
                console.log(`Technical Skills: ${details.ratings.technical_skills}/5`);
                console.log(`Overall Performance: ${details.overall_rating}/5`);
            }
        }
    }
}

    Best Practices¶

    Performance Evaluation¶

    1. Regular Reviews: Conduct formal appraisals at consistent intervals
    2. Continuous Feedback: Provide ongoing feedback throughout the period
    3. Objective Criteria: Use measurable performance indicators
    4. Documentation: Maintain detailed records of performance events
    5. Development Focus: Emphasize growth and improvement opportunities

    Data Management¶

    1. Confidentiality: Protect sensitive performance data
    2. Accuracy: Ensure ratings reflect actual performance
    3. Consistency: Apply rating standards uniformly
    4. Timeliness: Complete appraisals within scheduled timeframes
    5. Follow-up: Monitor implementation of development plans

    System Integration¶

    1. Training Records: Link appraisals to training completion
    2. Incident Reports: Consider safety incidents in evaluations
    3. Career Progression: Use appraisals for promotion decisions
    4. Compensation: Link performance to salary adjustments
    5. Succession Planning: Identify high-potential crew members

    Common Use Cases¶

    Annual Performance Reviews¶

    • Comprehensive evaluation of yearly performance
    • Goal setting for upcoming year
    • Career development planning
    • Compensation review preparation

    Probationary Assessments¶

    • Evaluate new hire performance
    • Confirm permanent employment status
    • Identify training needs
    • Early intervention for performance issues

    Promotion Evaluations¶

    • Assess readiness for advancement
    • Evaluate leadership potential
    • Review technical competencies
    • Compare candidates objectively

    Development Planning¶

    • Identify skill gaps
    • Create learning pathways
    • Track development progress
    • Measure training effectiveness

    Security and Compliance¶

    Data Protection¶

    • Encrypt sensitive performance data
    • Implement role-based access controls
    • Maintain audit logs of all changes
    • Regular backup of appraisal records

    Regulatory Compliance¶

    • Meet maritime industry standards
    • Support ISO certification requirements
    • Comply with employment regulations
    • Maintain documentation for inspections
