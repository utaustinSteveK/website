    GET /crew-members¶

    List crew members with optional filtering, sorting, and pagination.

    Endpoint¶

    GET /api/v1/crew-members

    Description¶

    This endpoint returns a paginated list of crew members with comprehensive filtering and search capabilities. Results can be filtered by various criteria including status, department, rank, nationality, vessel assignment, and certificate validity.

    Authentication¶

    Authentication required - Requires the crew.read permission.

    Allowed Roles: admin, crew_manager, hr_manager, fleet_manager

    Request¶

    Headers¶

    Authorization: Bearer <access_token>

    Query Parameters¶

    Pagination Parameters¶

    Parameter Type Required Default Description
    page integer No 1 Page number (minimum: 1)
    size integer No 20 Items per page (minimum: 1, maximum: 100)

    Filter Parameters¶

    Parameter Type Required Description
    status string No Filter by crew status (active, available, on_vessel, etc.)
    department string No Filter by department (deck, engine, catering, electrical, radio)
    rank string No Filter by crew rank/position
    vessel_id string No Filter by current vessel assignment
    nationality string No Filter by crew nationality
    has_valid_certificates boolean No Filter by certificate validity status
    search string No Search across name, email, and employee_id fields

    Date Filter Parameters¶

    Parameter Type Format Description
    joined_after string YYYY-MM-DD Filter by joining date (after)
    joined_before string YYYY-MM-DD Filter by joining date (before)
    cert_expiring_within_days integer - Filter crew with certificates expiring within specified days

    Sorting Parameters¶

    Parameter Type Description
    sort_by string Sort field (name, rank, date_joined, status)
    sort_order string Sort direction (asc, desc)

    Example Request URLs¶

    # Basic list request
GET /api/v1/crew-members?page=1&size=20

# Filter by status and department
GET /api/v1/crew-members?status=available&department=deck

# Search by name
GET /api/v1/crew-members?search=john&page=1&size=10

# Filter by certificate expiration
GET /api/v1/crew-members?cert_expiring_within_days=30

# Complex filter
GET /api/v1/crew-members?status=active&department=engine&nationality=Philippines&page=2&size=25

    Response¶

    Success Response (200 OK)¶

    Field Type Description
    success boolean Always true for successful responses
    data object Response data containing items and pagination
    data.items array Array of crew member objects
    data.pagination object Pagination information

    Pagination Object¶

    Field Type Description
    page integer Current page number
    size integer Items per page
    total integer Total number of items
    total_pages integer Total number of pages
    has_next boolean Whether there is a next page
    has_previous boolean Whether there is a previous page

    Crew Member Object¶

    Field Type Description
    id string Crew member UUID
    employee_id string Company employee identifier
    full_name string Full name (computed)
    first_name string First name
    last_name string Last name
    middle_name string Middle name (nullable)
    email string Email address
    phone string Primary phone number
    rank string Current rank/position
    department string Ship department
    status string Current operational status
    nationality string Nationality
    date_joined string Date joined company (ISO format)
    current_vessel_assignment object Current vessel assignment (nullable)
    certificate_status object Certificate validity summary
    created_at string Record creation timestamp
    updated_at string Last update timestamp

    Current Vessel Assignment Object¶

    Field Type Description
    id string Assignment UUID
    vessel_id string Vessel UUID
    vessel_name string Vessel name
    role string Role on vessel
    start_date string Assignment start date
    is_active boolean Whether assignment is active

    Certificate Status Object¶

    Field Type Description
    total_certificates integer Total number of certificates
    valid_certificates integer Number of valid certificates
    expiring_soon integer Certificates expiring within alert period
    expired_certificates integer Number of expired certificates
    compliance_score number Overall compliance score (0-100)

    Python¶

    import requests

def list_crew_members(access_token: str, filters: dict = None, page: int = 1, size: int = 20):
    """
    List crew members with optional filters.

    Args:
        access_token: Authentication token
        filters: Dictionary of filter parameters
        page: Page number
        size: Items per page

    Returns:
        dict: API response with crew members and pagination
    """
    url = "http://localhost:8000/api/v1/crew-members"
    headers = {
        "Authorization": f"Bearer {access_token}",
        "Content-Type": "application/json"
    }

    # Build query parameters
    params = {"page": page, "size": size}
    if filters:
        params.update(filters)

    try:
        response = requests.get(url, headers=headers, params=params, timeout=30)

        if response.status_code == 200:
            return response.json()
        else:
            error_data = response.json()
            print(f"API Error: {error_data.get('error', {}).get('message')}")
            return None

    except requests.RequestException as e:
        print(f"Network error: {e}")
        return None

# Usage examples
access_token = "your_access_token_here"

# Get first page of crew members
crew_list = list_crew_members(access_token)
if crew_list:
    print(f"Found {crew_list['data']['pagination']['total']} crew members")
    for crew in crew_list['data']['items']:
        print(f"- {crew['full_name']} ({crew['rank']}) - {crew['status']}")

# Filter by status and department
available_deck_crew = list_crew_members(
    access_token,
    filters={"status": "available", "department": "deck"},
    size=50
)

# Search for specific crew member
search_results = list_crew_members(
    access_token,
    filters={"search": "john"}
)

    import requests
from datetime import datetime, timedelta

class CrewMemberFilter:
    def __init__(self, access_token: str, base_url: str = "http://localhost:8000"):
        self.access_token = access_token
        self.base_url = base_url
        self.headers = {
            "Authorization": f"Bearer {access_token}",
            "Content-Type": "application/json"
        }

    def get_available_crew_by_department(self, department: str):
        """Get all available crew members in a specific department"""
        return self._list_crew({
            "status": "available",
            "department": department
        })

    def get_crew_with_expiring_certificates(self, days: int = 30):
        """Get crew members with certificates expiring soon"""
        return self._list_crew({
            "cert_expiring_within_days": days
        })

    def search_crew_by_name(self, name: str):
        """Search crew members by name"""
        return self._list_crew({"search": name})

    def get_crew_by_vessel(self, vessel_id: str):
        """Get crew members assigned to a specific vessel"""
        return self._list_crew({"vessel_id": vessel_id})

    def get_crew_joined_in_period(self, start_date: str, end_date: str):
        """Get crew members who joined within a date range"""
        return self._list_crew({
            "joined_after": start_date,
            "joined_before": end_date
        })

    def _list_crew(self, filters: dict, page: int = 1, size: int = 100):
        """Internal method to list crew with filters"""
        params = {"page": page, "size": size}
        params.update(filters)

        try:
            response = requests.get(
                f"{self.base_url}/api/v1/crew-members",
                headers=self.headers,
                params=params,
                timeout=30
            )

            if response.status_code == 200:
                return response.json()
            return None

        except requests.RequestException:
            return None

    JavaScript/TypeScript¶

    class CrewMemberAPI {
    constructor(baseUrl, accessToken) {
        this.baseUrl = baseUrl;
        this.headers = {
            'Authorization': `Bearer ${accessToken}`,
            'Content-Type': 'application/json'
        };
    }

    async listCrewMembers(options = {}) {
        const { filters = {}, page = 1, size = 20, sortBy, sortOrder } = options;

        // Build query parameters
        const params = new URLSearchParams({
            page: page.toString(),
            size: size.toString(),
            ...filters
        });

        if (sortBy) params.append('sort_by', sortBy);
        if (sortOrder) params.append('sort_order', sortOrder);

        try {
            const response = await fetch(
                `${this.baseUrl}/api/v1/crew-members?${params}`,
                { headers: this.headers }
            );

            if (response.ok) {
                return await response.json();
            } else {
                const errorData = await response.json();
                console.error('API Error:', errorData.error.message);
                return null;
            }
        } catch (error) {
            console.error('Network error:', error);
            return null;
        }
    }

    async getAllCrewMembers(filters = {}) {
        """Fetch all crew members by iterating through pages"""
        const allCrewMembers = [];
        let page = 1;
        let hasMore = true;

        while (hasMore) {
            const response = await this.listCrewMembers({
                filters,
                page,
                size: 100
            });

            if (response && response.data) {
                allCrewMembers.push(...response.data.items);
                hasMore = response.data.pagination.has_next;
                page++;
            } else {
                break;
            }
        }

        return allCrewMembers;
    }
}

    import { useState, useEffect } from 'react';

interface CrewMember {
    id: string;
    full_name: string;
    rank: string;
    department: string;
    status: string;
    // ... other fields
}

interface CrewListFilter {
    status?: string;
    department?: string;
    search?: string;
    nationality?: string;
}

interface UseCrewListOptions {
    filters?: CrewListFilter;
    page?: number;
    size?: number;
    autoFetch?: boolean;
}

export function useCrewList(options: UseCrewListOptions = {}) {
    const { filters = {}, page = 1, size = 20, autoFetch = true } = options;

    const [crewMembers, setCrewMembers] = useState<CrewMember[]>([]);
    const [loading, setLoading] = useState(false);
    const [error, setError] = useState<string | null>(null);
    const [pagination, setPagination] = useState({
        page: 1,
        size: 20,
        total: 0,
        total_pages: 0,
        has_next: false,
        has_previous: false
    });

    const fetchCrewMembers = async (newFilters?: CrewListFilter, newPage?: number) => {
        setLoading(true);
        setError(null);

        try {
            const currentFilters = newFilters || filters;
            const currentPage = newPage || page;

            const params = new URLSearchParams({
                page: currentPage.toString(),
                size: size.toString(),
                ...currentFilters
            });

            const response = await fetch(
                `/api/v1/crew-members?${params}`,
                {
                    headers: {
                        'Authorization': `Bearer ${getAccessToken()}`,
                        'Content-Type': 'application/json'
                    }
                }
            );

            if (response.ok) {
                const data = await response.json();
                setCrewMembers(data.data.items);
                setPagination(data.data.pagination);
            } else {
                const errorData = await response.json();
                setError(errorData.error.message);
            }
        } catch (err) {
            setError('Failed to fetch crew members');
            console.error('Fetch error:', err);
        } finally {
            setLoading(false);
        }
    };

    useEffect(() => {
        if (autoFetch) {
            fetchCrewMembers();
        }
    }, [JSON.stringify(filters), page, size, autoFetch]);

    return {
        crewMembers,
        loading,
        error,
        pagination,
        refetch: fetchCrewMembers,
        nextPage: () => pagination.has_next && fetchCrewMembers(filters, pagination.page + 1),
        previousPage: () => pagination.has_previous && fetchCrewMembers(filters, pagination.page - 1)
    };
}

// Usage in component
function CrewMemberList() {
    const [filters, setFilters] = useState({});

    const {
        crewMembers,
        loading,
        error,
        pagination,
        refetch,
        nextPage,
        previousPage
    } = useCrewList({ filters });

    const handleFilterChange = (newFilters) => {
        setFilters(newFilters);
    };

    if (loading) return <div>Loading...</div>;
    if (error) return <div>Error: {error}</div>;

    return (
        <div>
            <CrewFilters onFiltersChange={handleFilterChange} />

            <div className="crew-list">
                {crewMembers.map(crew => (
                    <div key={crew.id} className="crew-card">
                        <h3>{crew.full_name}</h3>
                        <p>{crew.rank} - {crew.department}</p>
                        <span className={`status ${crew.status}`}>
                            {crew.status}
                        </span>
                    </div>
                ))}
            </div>

            <div className="pagination">
                <button onClick={previousPage} disabled={!pagination.has_previous}>
                    Previous
                </button>
                <span>
                    Page {pagination.page} of {pagination.total_pages}
                </span>
                <button onClick={nextPage} disabled={!pagination.has_next}>
                    Next
                </button>
            </div>
        </div>
    );
}

    Notes¶

    Performance Considerations¶

    1. Use appropriate page sizes - larger pages reduce API calls but increase response time
    2. Apply filters to reduce result sets and improve performance
    3. Consider caching frequently accessed crew lists
    4. Use search wisely - specific searches are more efficient than broad queries

    Filtering Best Practices¶

    1. Combine filters for more precise results
    2. Use status filters to get actionable crew lists (e.g., available crew)
    3. Certificate expiration filtering helps with compliance monitoring
    4. Date range filtering is useful for reporting and analytics

    Error Handling¶

    1. Validate filter parameters before sending requests
    2. Handle pagination edge cases (empty results, invalid page numbers)
    3. Implement retry logic for network failures
    4. Display user-friendly error messages for validation failures

    Integration Tips¶

    1. Implement client-side caching for better user experience
    2. Use real-time updates when crew status changes frequently
    3. Combine with other APIs (vessels, certificates) for comprehensive views
    4. Export capabilities for reporting and data analysis
