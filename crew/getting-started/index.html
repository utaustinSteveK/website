    Getting Started¶

    Welcome to the Crew Management API! This section will help you get up and running quickly.

    Overview¶

    The Crew Management API provides comprehensive backend services for maritime crew management, including:

    • Authentication & Authorization - Secure JWT-based access control
    • Crew Data Management - Complete CRUD operations for crew members
    • Performance Appraisals - Structured evaluation workflows
    • Vessel Assignments - Fleet and crew rotation management
    • Certificate Tracking - Compliance and expiry monitoring
    • AI/LLM Integration - MCP tools for automation

    Prerequisites¶

    Required Software¶

    • Python 3.11+ - Primary backend language
    • PostgreSQL 15+ - Main database
    • Redis 7.0+ - Caching and sessions (optional)
    • Docker - For containerized deployment (optional)

    Recommended Tools¶

    • Postman or Insomnia - API testing
    • pgAdmin or DBeaver - Database management
    • Git - Version control
    • VS Code or PyCharm - IDE

    Architecture Overview¶

    sequenceDiagram
    participant C as Client
    participant A as API Gateway
    participant Auth as Auth Service
    participant DB as PostgreSQL
    participant Cache as Redis

    C->>A: Request + JWT
    A->>Auth: Validate Token
    Auth->>Cache: Check Session
    Cache-->>Auth: Session Valid
    Auth-->>A: Authorized
    A->>DB: Query Data
    DB-->>A: Return Results
    A-->>C: JSON Response

    Development Workflow¶

    1. Local Development¶

    # Clone repository
git clone https://github.com/example/crewing-backend.git
cd crewing-backend/backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt
pip install -r requirements-dev.txt

# Run development server
uvicorn src.main:app --reload

    2. Docker Development¶

    # Navigate to docker directory
cd backend/docker

# Start services
docker-compose up -d

# View logs
docker-compose logs -f backend

# Access at http://localhost:8000

    3. Testing¶

    # Run all tests
pytest tests/ -v

# Run with coverage
pytest --cov=src --cov-report=html

# Run specific test category
pytest tests/unit/ -v
pytest tests/integration/ -v
pytest tests/contract/ -v

    API Base URLs¶

    Environment Base URL Purpose
    Local http://localhost:8000 Local development
    Docker http://localhost:8000 Containerized local
    Staging https://api-staging.crewing.example.com Testing environment
    Production https://api.crewing.example.com Live environment

    Response Format¶

    All API responses follow a consistent JSON structure:

    Success Response¶

    {
  "status": "success",
  "data": {
    // Response data here
  },
  "meta": {
    "timestamp": "2024-01-15T10:30:00Z",
    "version": "1.0.0"
  }
}

    Error Response¶

    {
  "status": "error",
  "error": {
    "code": "AUTH_001",
    "message": "Invalid credentials",
    "details": {
      // Additional error context
    }
  },
  "meta": {
    "timestamp": "2024-01-15T10:30:00Z",
    "request_id": "req_12345"
  }
}

    Rate Limiting¶

    The API implements rate limiting to ensure fair usage:

    Endpoint Type Limit Window
    Authentication 5 requests 1 minute
    Read Operations 100 requests 1 minute
    Write Operations 50 requests 1 minute
    Bulk Operations 10 requests 1 minute

    Common Issues¶

    Database Connection

    If you encounter database connection errors, ensure PostgreSQL is running and the connection string in .env is correct.

    Port Already in Use

    If port 8000 is already in use, you can specify a different port: 

    uvicorn src.main:app --reload --port 8001

    Windows Users

    On Windows, use venv\Scripts\activate to activate the virtual environment and ensure PostgreSQL is installed via the official installer.
