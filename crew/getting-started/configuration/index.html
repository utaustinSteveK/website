    Configuration Guide¶

    This guide covers all configuration options for the Crew Appraisals Backend API. The application uses environment variables and Pydantic Settings for configuration management.

    Configuration Overview¶

    The application configuration is managed through:

    • Environment variables: Primary configuration method
    • .env file: Local development configuration
    • Default values: Built-in fallbacks for development

    All configuration is defined in src/core/config.py using Pydantic Settings.

    Environment File Setup¶

    Creating the Environment File¶

    # Copy the example file
cp .env.example .env

# Edit with your preferred editor
nano .env  # or vim .env, code .env

    Environment File Structure¶

    # Database Configuration
DATABASE_URL=postgresql+asyncpg://crew_user:crew_pass@localhost:5432/crew_appraisals
DATABASE_POOL_SIZE=20
DATABASE_MAX_OVERFLOW=40
DATABASE_POOL_TIMEOUT=30

# Redis Configuration
REDIS_URL=redis://localhost:6379/0
REDIS_POOL_SIZE=10

# Security
SECRET_KEY=dev-secret-key-change-in-production-to-something-really-secret
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=30
SESSION_EXPIRE_HOURS=24

# FastAPI Configuration
APP_NAME=Crew Appraisals Backend
APP_VERSION=1.0.0
API_V1_PREFIX=/api/v1
DEBUG=True
ENVIRONMENT=development
LOG_LEVEL=DEBUG

# CORS Configuration
CORS_ORIGINS=http://localhost:3000,http://localhost:5173,http://localhost:8080
CORS_ALLOW_CREDENTIALS=True
CORS_ALLOW_METHODS=*
CORS_ALLOW_HEADERS=*

# Additional configuration sections...

    Configuration Categories¶

    Database Configuration¶

    Configure PostgreSQL connection and pool settings.

    Variable Default Description
    DATABASE_URL postgresql+asyncpg://crew_user:crew_pass@localhost:5432/crew_appraisals PostgreSQL connection string
    DATABASE_POOL_SIZE 20 Connection pool size
    DATABASE_MAX_OVERFLOW 40 Maximum overflow connections
    DATABASE_POOL_TIMEOUT 30 Pool timeout in seconds

    Database URL Format¶

    # Standard format
DATABASE_URL=postgresql+asyncpg://username:password@host:port/database

# Examples for different environments
DATABASE_URL=postgresql+asyncpg://crew_user:crew_pass@localhost:5432/crew_appraisals  # Local
DATABASE_URL=postgresql+asyncpg://user:pass@db.example.com:5432/prod_db  # Production
DATABASE_URL=postgresql+asyncpg://user:pass@postgres:5432/crew_appraisals  # Docker

    Database Security

    Never commit database credentials to version control. Use environment-specific configurations.

    Connection Pool Tuning¶

    For production environments, adjust pool settings based on your workload:

    # High traffic applications
DATABASE_POOL_SIZE=50
DATABASE_MAX_OVERFLOW=100
DATABASE_POOL_TIMEOUT=60

# Low traffic applications
DATABASE_POOL_SIZE=5
DATABASE_MAX_OVERFLOW=10
DATABASE_POOL_TIMEOUT=10

    Redis Configuration¶

    Configure Redis for session management and caching.

    Variable Default Description
    REDIS_URL redis://localhost:6379/0 Redis connection URL
    REDIS_POOL_SIZE 10 Redis connection pool size

    Redis URL Examples¶

    # Local Redis
REDIS_URL=redis://localhost:6379/0

# Redis with password
REDIS_URL=redis://:password@localhost:6379/0

# Redis cluster
REDIS_URL=redis://user:pass@redis.example.com:6379/1

# Redis over SSL
REDIS_URL=rediss://user:pass@redis.example.com:6380/0

# Docker Redis
REDIS_URL=redis://redis:6379/0

    Redis Optional

    Redis is optional but recommended. The application will work without Redis, but sessions will be stored in the database only.

    Security Configuration¶

    Critical security settings for authentication and authorization.

    Variable Default Description
    SECRET_KEY dev-secret-key-change-in-production JWT signing key
    ALGORITHM HS256 JWT algorithm
    ACCESS_TOKEN_EXPIRE_MINUTES 30 Access token lifetime
    REFRESH_TOKEN_EXPIRE_DAYS 7 Refresh token lifetime
    SESSION_EXPIRE_HOURS 24 Session lifetime

    Generating Secure Keys¶

    Production Security

    ALWAYS change the SECRET_KEY in production environments!

    Generate a secure secret key:

    # Method 1: Using Python
python -c "import secrets; print(secrets.token_urlsafe(32))"

# Method 2: Using OpenSSL
openssl rand -hex 32

# Method 3: Using /dev/urandom
head -c 32 /dev/urandom | base64

    Token Lifetime Configuration¶

    # Short-lived tokens (high security)
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=1
SESSION_EXPIRE_HOURS=8

# Long-lived tokens (convenience)
ACCESS_TOKEN_EXPIRE_MINUTES=60
REFRESH_TOKEN_EXPIRE_DAYS=30
SESSION_EXPIRE_HOURS=72

    CORS Configuration¶

    Cross-Origin Resource Sharing settings for frontend integration.

    Variable Default Description
    CORS_ORIGINS http://localhost:3000,http://localhost:5173 Allowed origins
    CORS_ALLOW_CREDENTIALS True Allow credentials
    CORS_ALLOW_METHODS * Allowed HTTP methods
    CORS_ALLOW_HEADERS * Allowed headers

    CORS Examples¶

    # Development (multiple frontends)
CORS_ORIGINS=http://localhost:3000,http://localhost:5173,http://localhost:8080

# Production (specific domains)
CORS_ORIGINS=https://app.example.com,https://admin.example.com

# Staging environment
CORS_ORIGINS=https://staging.example.com,https://preview.example.com

    CORS Security

    In production, specify exact domains instead of using wildcards (*).

    MCP (Model Context Protocol) Configuration¶

    Configure FASTMCP integration for AI/LLM features.

    Variable Default Description
    MCP_ENABLED True Enable MCP integration
    MCP_PATH /mcp MCP endpoint path
    MCP_NAME crew-appraisals-mcp MCP server name
    MCP_DESCRIPTION LLM interface for crew management system MCP description
    MCP_VERSION 1.0.0 MCP version

    Disabling MCP¶

    To disable MCP features:

    MCP_ENABLED=False

    Tune application performance settings.

    Variable Default Description
    MAX_WORKERS 4 Maximum worker processes
    KEEPALIVE 5 Keep-alive timeout
    REQUEST_TIMEOUT 60 Request timeout seconds

REQUEST_TIMEOUT=120

# Low-resource server (2 cores)
MAX_WORKERS=2
KEEPALIVE=5
REQUEST_TIMEOUT=30

# Development (single worker for debugging)
MAX_WORKERS=1
KEEPALIVE=5
REQUEST_TIMEOUT=60

    Monitoring Configuration¶

    Configure metrics and tracing.

    Variable Default Description
    ENABLE_METRICS True Enable Prometheus metrics
    METRICS_PATH /metrics Metrics endpoint
    ENABLE_TRACING False Enable distributed tracing
    JAEGER_AGENT_HOST localhost Jaeger agent host
    JAEGER_AGENT_PORT 6831 Jaeger agent port

ENABLE_METRICS=True
METRICS_PATH=/metrics
ENABLE_TRACING=True
JAEGER_AGENT_HOST=jaeger
JAEGER_AGENT_PORT=6831

    Rate Limiting Configuration¶

    Configure API rate limiting.

    Variable Default Description
    RATE_LIMIT_ENABLED True Enable rate limiting
    RATE_LIMIT_PER_MINUTE 60 Requests per minute
    RATE_LIMIT_BURST 10 Burst limit

    Rate Limiting Examples¶

    # Strict rate limiting
RATE_LIMIT_ENABLED=True
RATE_LIMIT_PER_MINUTE=30
RATE_LIMIT_BURST=5

# Relaxed rate limiting
RATE_LIMIT_ENABLED=True
RATE_LIMIT_PER_MINUTE=120
RATE_LIMIT_BURST=20

# Disable rate limiting (not recommended)
RATE_LIMIT_ENABLED=False

    Logging Configuration¶

    The application supports structured logging with different levels:

    # Logging levels (from most to least verbose)
LOG_LEVEL=DEBUG    # Development debugging
LOG_LEVEL=INFO     # General information
LOG_LEVEL=WARNING  # Warning messages only
LOG_LEVEL=ERROR    # Error messages only
LOG_LEVEL=CRITICAL # Critical errors only

    Custom Logging Configuration¶

    You can also configure structured logging in the application:

    # In your startup script or custom configuration
import logging
import sys

logging.basicConfig(
    level=getattr(logging, os.getenv('LOG_LEVEL', 'INFO')),
    format='%(asctime)s - %(name)s - %(levelname)s - %(message)s',
    handlers=[
        logging.StreamHandler(sys.stdout),
        logging.FileHandler('/var/log/crew-api.log')  # Optional file logging
    ]
)

    Configuration Validation¶

    The application validates configuration on startup. Common validation errors:

    Invalid Database URL¶

    ValueError: Invalid database URL format
    Solution: Ensure the DATABASE_URL follows the correct format: 
    DATABASE_URL=postgresql+asyncpg://username:password@host:port/database

    Missing Secret Key¶

    ValueError: SECRET_KEY must be provided in production
    Solution: Set a secure secret key: 
    SECRET_KEY=$(python -c "import secrets; print(secrets.token_urlsafe(32))")

    Invalid CORS Origins¶

    ValueError: Invalid CORS origin format
    Solution: Ensure CORS origins are valid URLs: 
    CORS_ORIGINS=https://app.example.com,https://admin.example.com

    Configuration Best Practices¶

    1. Use Environment-Specific Files: 

      .env.development
.env.staging
.env.production

    2. Never Commit Secrets: 

      # Add to .gitignore
.env*
!.env.example

    3. Use Configuration Management:

    4. Docker secrets for containers
    5. Kubernetes ConfigMaps and Secrets

    6. Cloud provider parameter stores (AWS Parameter Store, Azure Key Vault)

    7. Validate Configuration: 

      # Test configuration before deployment
poetry run python -c "from src.core.config import settings; print('Config valid')"

    8. Document Custom Settings: Always document any custom environment variables you add.
