    Installation Guide¶

    This comprehensive guide covers installing and setting up the Crew Management Backend API for development and production environments.

    System Requirements¶

    Minimum Requirements (Docker Compose)¶

    • Docker: 20.10 or higher
    • Docker Compose: v2.0 or higher
    • Memory: 4GB RAM minimum
    • Storage: 10GB available space
    • OS: Linux, macOS, or Windows 10/11 (with Docker Desktop)

    Requirements for Python Development (Optional)¶

    • Python: 3.11 or higher
    • Database: PostgreSQL 12 or higher
    • Cache: Redis 6 or higher (optional)
    • Memory: 8GB RAM for development
    • CPU: 2+ cores

    Installation Methods¶

    Choose the installation method that best fits your needs:

    Best for quick setup, consistent environments, and production deployment.

    For Python development and customization.

    For custom setups or learning purposes.

    2. Open the .dmg file and drag Docker to Applications

    Step 2: Clone Repository and Setup¶

    # Clone repository
git clone https://github.com/example/crewing-backend.git
cd crewing-backend

    # Start all services in detached mode

    Docker Compose Services¶

    The Docker Compose setup includes:

    Service Port Purpose
    backend 8000 FastAPI application server
    postgres 5432 PostgreSQL database
    redis 6379 Cache and session storage
    nginx 80 Reverse proxy and load balancer

    Managing Docker Services¶

    # Stop all services
docker-compose down

# Stop and remove all data
docker-compose down -v

# Restart a specific service
docker-compose restart backend

# View service logs
docker-compose logs -f [service_name]

# Execute commands in container
docker-compose exec backend /bin/bash

    Database Setup¶

    PostgreSQL Installation¶

    # Install PostgreSQL
sudo apt install postgresql postgresql-contrib

# Start PostgreSQL service
sudo systemctl start postgresql
sudo systemctl enable postgresql

# Create database and user
sudo -u postgres psql

    CREATE USER crew_user WITH PASSWORD 'crew_pass';
CREATE DATABASE crew_appraisals OWNER crew_user;
GRANT ALL PRIVILEGES ON DATABASE crew_appraisals TO crew_user;
\q

    # Install PostgreSQL via Homebrew
brew install postgresql

# Start PostgreSQL service
brew services start postgresql

# Create database and user
createuser -s crew_user
createdb -O crew_user crew_appraisals

# Set password
psql -d crew_appraisals -c "ALTER USER crew_user PASSWORD 'crew_pass';"

    # Start PostgreSQL container
docker run -d \
  --name postgres-crew \
  -e POSTGRES_DB=crew_appraisals \
  -e POSTGRES_USER=crew_user \
  -e POSTGRES_PASSWORD=crew_pass \
  -p 5432:5432 \
  postgres:15

# Or use the provided docker-compose file
cd docker
docker-compose -f docker-compose.dev.yaml up -d postgres

    Database Configuration¶

    1. Verify Connection: 

      # Test PostgreSQL connection
psql -h localhost -U crew_user -d crew_appraisals -c "SELECT version();"

    2. Run Migrations: 

      # Initialize database schema
poetry run alembic upgrade head

    Environment Configuration¶

    Step 1: Copy Environment File¶

    # Copy the example environment file
cp .env.example .env

    Step 2: Configure Environment Variables¶

    Edit the .env file with your specific settings:

    # Database Configuration
DATABASE_URL=postgresql+asyncpg://crew_user:crew_pass@localhost:5432/crew_appraisals

# Redis Configuration (if using)
REDIS_URL=redis://localhost:6379/0

# Security - IMPORTANT: Change in production
SECRET_KEY=your-super-secret-key-here-change-in-production

# Environment
ENVIRONMENT=development  # or production
DEBUG=True  # Set to False in production
LOG_LEVEL=DEBUG  # Use INFO or WARNING in production

    Security Warning

    Always change the SECRET_KEY in production environments. Generate a secure key using: 

    python -c "import secrets; print(secrets.token_urlsafe(32))"

    Troubleshooting¶

    Common Issues¶

    Poetry command not found

    Solution: Add Poetry to your PATH or restart your terminal: 

    echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

    PostgreSQL connection refused

    Solutions: 

    # Check if PostgreSQL is running
sudo systemctl status postgresql

# Start PostgreSQL if stopped
sudo systemctl start postgresql

# Check connection settings in .env file
cat .env | grep DATABASE_URL

    Permission denied for database

    Solution: Ensure the database user has proper permissions: 

    sudo -u postgres psql
GRANT ALL PRIVILEGES ON DATABASE crew_appraisals TO crew_user;
GRANT ALL ON SCHEMA public TO crew_user;

    Redis connection error

    Solutions: 

    # Check Redis status
redis-cli ping

# Start Redis if stopped
sudo systemctl start redis-server

# Check Redis URL in .env
cat .env | grep REDIS_URL

    Python version issues

    Solution: Ensure you're using Python 3.11: 

    # Check active Python version
python --version

# Use specific Python version with Poetry
poetry env use python3.11

    Performance Tuning¶

    For production environments, consider these optimizations:

    1. Database Connection Pooling: 

      # Adjust in .env
DATABASE_POOL_SIZE=50
DATABASE_MAX_OVERFLOW=100

    2. Worker Processes: 

      # Adjust based on CPU cores
MAX_WORKERS=8

    3. Memory Settings: 

      # PostgreSQL memory settings
sudo nano /etc/postgresql/15/main/postgresql.conf
# Adjust shared_buffers, effective_cache_size

    Log Files¶

    Monitor these log files for issues:

    • Application logs: Check console output or configure file logging
    • PostgreSQL logs: /var/log/postgresql/postgresql-15-main.log
    • Redis logs: /var/log/redis/redis-server.log
    • Docker logs: docker compose logs -f
