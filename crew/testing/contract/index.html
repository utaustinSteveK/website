    Contract Testing Guide¶

    Contract testing validates the interfaces between services, ensuring that APIs meet their documented specifications and that consumers can safely interact with providers. In the Crew Appraisals System, contract testing focuses on OpenAPI specification validation and request/response format compliance.

    Overview¶

    Contract testing sits between unit and integration testing in our test pyramid, providing:

    • API Specification Compliance: Ensuring endpoints match OpenAPI documentation
    • Request/Response Validation: Verifying data formats and schemas
    • Backward Compatibility: Detecting breaking changes in APIs
    • Cross-Service Communication: Validating service interactions
    • Documentation Accuracy: Keeping API docs synchronized with implementation

    Contract Testing Strategy¶

    graph TB
    A[OpenAPI Specification] --> B[Contract Tests]
    B --> C[Request Validation]
    B --> D[Response Validation]
    B --> E[Schema Compliance]
    C --> F[Test Results]
    D --> F
    E --> F

    Provider-Side Testing¶

    • Validate that our API implementation matches the OpenAPI specification
    • Ensure all endpoints return responses in the documented format
    • Verify error responses follow the specified schemas

    Consumer-Side Testing¶

    • Test that client code can handle all documented response formats
    • Validate request generation matches API expectations
    • Ensure proper error handling for documented error responses

    OpenAPI Contract Validation¶

    OpenAPI Specification¶

    Our API specification serves as the contract between frontend and backend:

    # openapi.yaml (excerpt)
openapi: 3.0.3
info:
  title: Crew Appraisals API
  version: 1.0.0
  description: REST API for managing crew appraisals and performance reviews

paths:
  /api/v1/crew-members:
    post:
      summary: Create a new crew member
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/CrewMemberCreate'
      responses:
        '201':
          description: Crew member created successfully
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/CrewMember'
        '400':
          description: Validation error
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ValidationError'

components:
  schemas:
    CrewMemberCreate:
      type: object
      required:
        - name
        - email
        - rank
      properties:
        name:
          type: string
          minLength: 1
          maxLength: 100
        email:
          type: string
          format: email
        rank:
          type: string
          enum: [Able Seaman, Bosun, Chief Engineer, Captain]

    Contract Test Implementation¶

    # tests/contract/test_crew_members_contract.py
import pytest
from httpx import AsyncClient
from openapi_spec_validator import validate_spec
from openapi_core import create_spec
from openapi_core.validation.request.validators import RequestValidator
from openapi_core.validation.response.validators import ResponseValidator
import yaml

class TestCrewMembersContract:
    """Contract tests for crew members API."""

    @pytest.fixture
    def openapi_spec(self):
        """Load OpenAPI specification."""
        with open("openapi.yaml", "r") as f:
            spec_dict = yaml.safe_load(f)
        validate_spec(spec_dict)  # Validate spec is valid OpenAPI
        return create_spec(spec_dict)

    @pytest.mark.asyncio
    async def test_create_crew_member_request_contract(self, async_client, openapi_spec):
        """Test that create crew member request matches OpenAPI spec."""
        # Arrange
        request_data = {
            "name": "John Doe",
            "email": "john.doe@example.com",
            "rank": "Able Seaman"
        }

        # Act
        response = await async_client.post(
            "/api/v1/crew-members",
            json=request_data,
            headers={"Content-Type": "application/json"}
        )

        # Assert - Validate request against OpenAPI spec
        validator = RequestValidator(openapi_spec)
        request = self._build_openapi_request("POST", "/api/v1/crew-members", request_data)
        result = validator.validate(request)

        assert not result.errors, f"Request validation errors: {result.errors}"

    @pytest.mark.asyncio
    async def test_create_crew_member_response_contract(self, async_client, openapi_spec):
        """Test that create crew member response matches OpenAPI spec."""
        # Arrange
        request_data = {
            "name": "Jane Smith",
            "email": "jane.smith@example.com",
            "rank": "Bosun"
        }

        # Act
        response = await async_client.post("/api/v1/crew-members", json=request_data)

        # Assert - Validate response against OpenAPI spec
        validator = ResponseValidator(openapi_spec)
        openapi_response = self._build_openapi_response(response)
        request = self._build_openapi_request("POST", "/api/v1/crew-members", request_data)

        result = validator.validate(request, openapi_response)
        assert not result.errors, f"Response validation errors: {result.errors}"

        # Additional assertions for successful creation
        assert response.status_code == 201
        response_data = response.json()
        assert "id" in response_data
        assert response_data["name"] == request_data["name"]
        assert response_data["email"] == request_data["email"]

    @pytest.mark.asyncio
    async def test_create_crew_member_validation_error_contract(self, async_client, openapi_spec):
        """Test that validation errors match OpenAPI spec."""
        # Arrange - Invalid data
        request_data = {
            "name": "",  # Invalid: empty name
            "email": "invalid-email",  # Invalid: bad email format
            "rank": "Invalid Rank"  # Invalid: not in enum
        }

        # Act
        response = await async_client.post("/api/v1/crew-members", json=request_data)

        # Assert
        assert response.status_code == 400

        # Validate error response format
        validator = ResponseValidator(openapi_spec)
        openapi_response = self._build_openapi_response(response)
        request = self._build_openapi_request("POST", "/api/v1/crew-members", request_data)

        result = validator.validate(request, openapi_response)
        assert not result.errors, f"Error response validation errors: {result.errors}"

        # Verify error structure
        error_data = response.json()
        assert "detail" in error_data
        assert isinstance(error_data["detail"], list)

    def _build_openapi_request(self, method, path, data):
        """Build OpenAPI request object for validation."""
        from openapi_core.contrib.requests import RequestsOpenAPIRequest
        import requests

        # Create a requests.Request object
        req = requests.Request(
            method=method,
            url=f"http://test{path}",
            json=data,
            headers={"Content-Type": "application/json"}
        )
        prepared = req.prepare()
        return RequestsOpenAPIRequest(prepared)

    def _build_openapi_response(self, httpx_response):
        """Build OpenAPI response object for validation."""
        from openapi_core.contrib.requests import RequestsOpenAPIResponse
        import requests

        # Convert HTTPX response to requests-like response
        response = requests.Response()
        response.status_code = httpx_response.status_code
        response.headers.update(httpx_response.headers)
        response._content = httpx_response.content
        return RequestsOpenAPIResponse(response)

    Schema-Based Validation¶

    # tests/contract/test_schema_validation.py
import pytest
from pydantic import ValidationError
from src.schemas.crew_member import CrewMemberCreate, CrewMemberResponse

class TestSchemaValidation:
    """Test Pydantic schema validation against OpenAPI schemas."""

    def test_crew_member_create_schema_validation(self):
        """Test CrewMemberCreate schema validates correctly."""
        # Valid data
        valid_data = {
            "name": "Test User",
            "email": "test@example.com",
            "rank": "Able Seaman"
        }

        schema = CrewMemberCreate(**valid_data)
        assert schema.name == "Test User"
        assert schema.email == "test@example.com"
        assert schema.rank == "Able Seaman"

    def test_crew_member_create_schema_validation_errors(self):
        """Test CrewMemberCreate schema validation errors."""
        test_cases = [
            # Missing required fields
            ({"name": "Test"}, "email"),
            ({"email": "test@example.com"}, "name"),

            # Invalid data types
            ({"name": 123, "email": "test@example.com", "rank": "Able Seaman"}, "name"),

            # Invalid email format
            ({"name": "Test", "email": "invalid", "rank": "Able Seaman"}, "email"),

            # Invalid enum value
            ({"name": "Test", "email": "test@example.com", "rank": "Invalid"}, "rank"),

            # String length constraints
            ({"name": "", "email": "test@example.com", "rank": "Able Seaman"}, "name"),
            ({"name": "x" * 101, "email": "test@example.com", "rank": "Able Seaman"}, "name"),
        ]

        for invalid_data, expected_field in test_cases:
            with pytest.raises(ValidationError) as exc_info:
                CrewMemberCreate(**invalid_data)

            errors = exc_info.value.errors()
            error_fields = [error["loc"][0] for error in errors]
            assert expected_field in error_fields

    def test_crew_member_response_schema(self):
        """Test CrewMemberResponse schema."""
        response_data = {
            "id": 1,
            "name": "Test User",
            "email": "test@example.com",
            "rank": "Able Seaman",
            "created_at": "2024-01-01T00:00:00Z",
            "updated_at": "2024-01-01T00:00:00Z"
        }

        schema = CrewMemberResponse(**response_data)
        assert schema.id == 1
        assert schema.created_at is not None
        assert schema.updated_at is not None

    # tests/contract/test_json_schema.py
import json
import jsonschema
import pytest

class TestJSONSchemaValidation:
    """Test responses against JSON schemas derived from OpenAPI."""

    @pytest.fixture
    def crew_member_schema(self):
        """JSON schema for crew member response."""
        return {
            "type": "object",
            "required": ["id", "name", "email", "rank", "created_at"],
            "properties": {
                "id": {"type": "integer", "minimum": 1},
                "name": {"type": "string", "minLength": 1, "maxLength": 100},
                "email": {"type": "string", "format": "email"},
                "rank": {
                    "type": "string",
                    "enum": ["Able Seaman", "Bosun", "Chief Engineer", "Captain"]
                },
                "created_at": {"type": "string", "format": "date-time"},
                "updated_at": {"type": "string", "format": "date-time"}
            },
            "additionalProperties": false
        }

    @pytest.mark.asyncio
    async def test_crew_member_response_json_schema(self, async_client, crew_member_schema):
        """Test crew member response against JSON schema."""
        # Arrange
        request_data = {
            "name": "Schema Test User",
            "email": "schema.test@example.com",
            "rank": "Chief Engineer"
        }

        # Act
        response = await async_client.post("/api/v1/crew-members", json=request_data)

        # Assert
        assert response.status_code == 201

        response_data = response.json()
        try:
            jsonschema.validate(response_data, crew_member_schema)
        except jsonschema.ValidationError as e:
            pytest.fail(f"Response doesn't match JSON schema: {e.message}")

    def test_validation_error_schema(self):
        """Test validation error response schema."""
        error_schema = {
            "type": "object",
            "required": ["detail"],
            "properties": {
                "detail": {
                    "type": "array",
                    "items": {
                        "type": "object",
                        "required": ["type", "loc", "msg"],
                        "properties": {
                            "type": {"type": "string"},
                            "loc": {"type": "array"},
                            "msg": {"type": "string"}
                        }
                    }
                }
            }
        }

        # Example validation error response
        error_response = {
            "detail": [
                {
                    "type": "value_error",
                    "loc": ["body", "email"],
                    "msg": "invalid email format"
                }
            ]
        }

        jsonschema.validate(error_response, error_schema)

    # tests/contract/test_appraisals_contract.py
@pytest.mark.asyncio
async def test_create_appraisal_contract(async_client, openapi_spec):
    """Test appraisal creation contract."""
    # First create a crew member
    crew_member_data = {
        "name": "Test Appraisee",
        "email": "appraisee@example.com",
        "rank": "Able Seaman"
    }
    member_response = await async_client.post("/api/v1/crew-members", json=crew_member_data)
    member_id = member_response.json()["id"]

    # Create appraisal
    appraisal_data = {
        "appraisee_id": member_id,
        "appraiser_id": member_id,  # Self-appraisal for simplicity
        "review_period_start": "2024-01-01",
        "review_period_end": "2024-06-30",
        "overall_rating": 4,
        "technical_skills": 4,
        "communication": 3,
        "leadership": 4,
        "comments": "Good performance overall."
    }

    # Act
    response = await async_client.post("/api/v1/appraisals", json=appraisal_data)

    # Assert
    assert response.status_code == 201

    # Validate against OpenAPI contract
    response_data = response.json()
    expected_fields = [
        "id", "appraisee_id", "appraiser_id", "review_period_start",
        "review_period_end", "overall_rating", "status", "created_at"
    ]

    for field in expected_fields:
        assert field in response_data, f"Missing required field: {field}"

    # Validate data types and constraints
    assert isinstance(response_data["id"], int)
    assert response_data["overall_rating"] >= 1
    assert response_data["overall_rating"] <= 5
    assert response_data["status"] in ["draft", "submitted", "approved"]

    Pre-commit Contract Testing¶

    # .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: contract-tests
        name: contract-tests
        entry: pytest tests/contract/ -v
        language: system
        pass_filenames: false
        always_run: true

    OpenAPI Spec Generation¶

    Automatically generate and validate OpenAPI specs:

    # scripts/generate_openapi.py
from fastapi.openapi.utils import get_openapi
from src.main import app
import json

def generate_openapi_spec():
    """Generate OpenAPI specification from FastAPI app."""
    openapi_schema = get_openapi(
        title="Crew Appraisals API",
        version="1.0.0",
        description="REST API for managing crew appraisals and performance reviews",
        routes=app.routes,
    )

    with open("openapi.json", "w") as f:
        json.dump(openapi_schema, f, indent=2)

    print("OpenAPI specification generated: openapi.json")

if __name__ == "__main__":
    generate_openapi_spec()

    Contract Testing in CI/CD¶

    # .github/workflows/contract-tests.yml
name: Contract Tests

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  contract-tests:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'

      - name: Install dependencies
        run: |
          poetry install

      - name: Run contract tests
        run: |
          poetry run pytest tests/contract/ -v --tb=short

      - name: Validate OpenAPI spec
        run: |
          poetry run python scripts/generate_openapi.py
          poetry run swagger-codegen validate -i openapi.json

    Best Practices¶

    1. Keep Contracts Simple¶

    • Focus on interface contracts, not implementation details
    • Use clear, consistent naming conventions
    • Document expected behavior explicitly

    2. Version Your Contracts¶

    • Use semantic versioning for API contracts
    • Maintain backward compatibility when possible
    • Clearly document breaking changes

    3. Automate Contract Validation¶

    • Run contract tests in CI/CD pipelines
    • Generate contracts from code when possible
    • Validate contracts against live APIs

    4. Test Edge Cases¶

    • Test all documented error scenarios
    • Validate optional fields and nullable values
    • Test boundary conditions (min/max values)

    5. Monitor Contract Changes¶

    • Track contract evolution over time
    • Alert on breaking changes
    • Review contract modifications in PRs

    Contract testing ensures that our API remains reliable and predictable, making it easier for frontend developers and external integrators to work with our system confidently.
