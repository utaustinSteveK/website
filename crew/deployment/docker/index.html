    Docker Deployment Guide¶

    This guide covers deploying the Crew Appraisals System using Docker and Docker Compose, which provides a consistent, containerized environment for development, testing, and production deployments.

    Overview¶

    The Docker setup includes:

    • Multi-stage Dockerfile for optimized builds
    • Docker Compose configurations for different environments
    • Service orchestration with health checks and dependencies
    • Development tools integration (pgAdmin, monitoring)
    • Production optimizations (distroless images, resource limits)

    Prerequisites¶

    System Requirements¶

    • Docker: Version 20.10+
    • Docker Compose: Version 2.0+
    • Memory: Minimum 4GB RAM (8GB recommended for development)
    • Storage: 10GB available space
    • Network: Ports 8000, 5432, 6379, 5050 available

    Installation¶

    Linux (Ubuntu/Debian)¶

    # Install Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Install Docker Compose
sudo apt-get install docker-compose-plugin

# Add user to docker group
sudo usermod -aG docker $USER
newgrp docker

    macOS¶

    # Install Docker Desktop
brew install --cask docker

# Verify installation
docker --version
docker compose version

    Windows¶

    1. Install Docker Desktop from docker.com
    2. Enable WSL2 backend
    3. Restart system

    Verification¶

    # Test Docker installation

# Test Docker Compose

    Docker Configuration Files¶

    Dockerfile Analysis¶

    The multi-stage Dockerfile provides optimized builds for different use cases:

    # Stage 1: Base image with Python and Poetry
FROM python:3.11-slim as base
ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    POETRY_VERSION=1.6.1

# Stage 2: Dependencies installation
FROM base as dependencies
COPY pyproject.toml poetry.lock* ./
RUN poetry install --no-root --no-dev

# Stage 3: Development (includes dev dependencies)
FROM dependencies as development
RUN poetry install --no-root
COPY . .
CMD ["uvicorn", "src.main:app", "--host", "0.0.0.0", "--port", "8000", "--reload"]

# Stage 4: Testing (includes test dependencies)
FROM dependencies as testing
RUN poetry install --no-root
COPY . .
CMD ["pytest", "-v", "--cov=src", "--cov-report=term-missing"]

# Stage 5: Production (distroless, optimized)
FROM gcr.io/distroless/python3.11-debian12 as production
COPY --from=production-build /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=production-build /app /app
ENTRYPOINT ["python", "-m", "uvicorn", "src.main:app"]
CMD ["--host", "0.0.0.0", "--port", "8000", "--workers", "4"]

    Core Services¶

    PostgreSQL Database: 

    postgres:
  image: postgres:15-alpine
  environment:
    POSTGRES_USER: ${DB_USER:-crewing_user}
    POSTGRES_PASSWORD: ${DB_PASSWORD:-crewing_pass}
    POSTGRES_DB: ${DB_NAME:-crew_appraisals}
  volumes:
    - postgres_data:/var/lib/postgresql/data
    - ./postgres/init.sql:/docker-entrypoint-initdb.d/01-init.sql:ro

    test: ["CMD-SHELL", "pg_isready -U ${DB_USER:-crewing_user}"]

    Redis Cache:

  image: redis:7-alpine
  command: redis-server --appendonly yes --requirepass ${REDIS_PASSWORD:-redis_pass}

  healthcheck:
    test: ["CMD", "redis-cli", "--raw", "incr", "ping"]

    Backend API: 

    backend:
  build:
    context: ../
    dockerfile: docker/Dockerfile
    target: ${BUILD_TARGET:-development}
  environment:
    DATABASE_URL: postgresql+asyncpg://${DB_USER}:${DB_PASSWORD}@postgres:5432/${DB_NAME}

  depends_on:
    postgres:
      condition: service_healthy

      condition: service_healthy

    Deployment Scenarios¶

    Local Development¶

    Purpose: Day-to-day development with hot reloading

    # Navigate to docker directory
cd backend/docker

# Start development environment
docker compose --profile development up -d

# View logs
docker compose logs -f backend

# Stop services
docker compose down

    Development Features: - Hot reloading enabled - pgAdmin for database management - Debug logging - Source code mounted for instant updates

    Access URLs: - API: http://localhost:8000 - API Documentation: http://localhost:8000/docs - pgAdmin: http://localhost:5050 - Health Check: http://localhost:8000/api/v1/health

    Testing Environment¶

    Purpose: Automated testing in CI/CD

    # Run tests in Docker
docker compose run --rm backend pytest

# Run with coverage
docker compose run --rm backend pytest --cov=src --cov-report=term-missing

# Run specific test category
docker compose run --rm backend pytest tests/unit/

# Build testing target
docker compose build --target testing backend
docker compose run --rm backend

    Staging Environment¶

    Purpose: Production-like testing environment

    # Use production compose file with staging overrides
docker compose -f docker-compose.yaml -f docker-compose.prod.yaml up -d

    Database Management¶

    Database Initialization¶

    The PostgreSQL service includes initialization scripts:

    -- docker/postgres/init.sql
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
CREATE EXTENSION IF NOT EXISTS "pg_trgm";

-- Create application user (if not exists)
DO $$
BEGIN
   IF NOT EXISTS (SELECT FROM pg_catalog.pg_roles WHERE rolname = 'app_user') THEN
      CREATE ROLE app_user WITH LOGIN PASSWORD 'app_password';
   END IF;
END
$$;

-- Grant necessary permissions
GRANT CONNECT ON DATABASE crew_appraisals TO app_user;
GRANT USAGE ON SCHEMA public TO app_user;
GRANT CREATE ON SCHEMA public TO app_user;

    Database Operations¶

    # Run migrations
docker compose exec backend alembic upgrade head

# Create new migration
docker compose exec backend alembic revision --autogenerate -m "Add new table"

# Connect to database
docker compose exec postgres psql -U crewing_user -d crew_appraisals

# Backup database
docker compose exec postgres pg_dump -U crewing_user crew_appraisals > backup.sql

# Restore database
cat backup.sql | docker compose exec -T postgres psql -U crewing_user crew_appraisals

    Production Optimizations¶

    Security Hardening¶

    # docker-compose.prod.yaml
services:
  postgres:
    # Remove exposed ports in production
    # ports: ["5432:5432"]  # Commented out
    networks:
      - backend_network  # Internal network only

  backend:
    # Use production image
    build:
      target: production
    # Set resource limits
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
    # Security options
    security_opt:
      - no-new-privileges:true
    read_only: true
    tmpfs:
      - /tmp

    SSL/TLS with Nginx¶

    # docker/nginx/conf.d/default.conf
server {
    listen 80;
    listen [::]:80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/fullchain.pem;
    ssl_certificate_key /etc/nginx/ssl/privkey.pem;

    # Security headers
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
    add_header X-Content-Type-Options nosniff always;
    add_header X-Frame-Options DENY always;

    location / {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

    Backup and Recovery¶

    Automated Backups¶

    #!/bin/bash
# docker/scripts/backup.sh

# Configuration
BACKUP_DIR="/backups"
RETENTION_DAYS=7
TIMESTAMP=$(date +%Y%m%d_%H%M%S)

# Create backup directory
mkdir -p $BACKUP_DIR

# Database backup
echo "Creating database backup..."
docker compose exec -T postgres pg_dump \
    -U crewing_user \
    -d crew_appraisals \
    --clean \
    --no-owner \
    --no-privileges > "$BACKUP_DIR/db_backup_$TIMESTAMP.sql"

# Compress backup
gzip "$BACKUP_DIR/db_backup_$TIMESTAMP.sql"

# Redis backup
echo "Creating Redis backup..."
docker compose exec -T redis redis-cli \
    --rdb /data/dump_$TIMESTAMP.rdb SAVE
docker cp $(docker compose ps -q redis):/data/dump_$TIMESTAMP.rdb "$BACKUP_DIR/"
gzip "$BACKUP_DIR/dump_$TIMESTAMP.rdb"

# Cleanup old backups
find "$BACKUP_DIR" -name "*.gz" -mtime +$RETENTION_DAYS -delete

echo "Backup completed: $TIMESTAMP"

    Restore Procedures¶

    #!/bin/bash
# docker/scripts/restore.sh

BACKUP_FILE=$1
if [ -z "$BACKUP_FILE" ]; then
    echo "Usage: $0 <backup_file.sql.gz>"
    exit 1
fi

# Stop application
echo "Stopping application..."
docker compose stop backend

# Restore database
echo "Restoring database..."
zcat "$BACKUP_FILE" | docker compose exec -T postgres psql \
    -U crewing_user \
    -d crew_appraisals

# Restart application
echo "Starting application..."
docker compose start backend

# Verify health
sleep 10
curl -f http://localhost:8000/api/v1/health

echo "Restore completed!"

    CI/CD Integration¶

    GitHub Actions Example¶

    # .github/workflows/docker.yml
name: Docker Build and Test

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v3

      - name: Build test image
        run: |
          cd backend
          docker compose build --target testing backend

      - name: Run tests
        run: |
          cd backend
          docker compose run --rm backend pytest --cov=src

      - name: Integration tests
        run: |
          cd backend
          docker compose --profile testing up -d
          docker compose run --rm backend pytest tests/integration/
          docker compose down

  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'

    steps:
      - uses: actions/checkout@v3

      - name: Deploy to staging
        run: |
          cd backend
          docker compose -f docker-compose.prod.yaml up -d

      - name: Health check
        run: |
          sleep 30
          curl -f http://localhost:8000/api/v1/health

    Best Practices¶

    Development Workflow¶

    1. Use profiles: Organize services with Docker Compose profiles
    2. Environment consistency: Use the same base images across environments
    3. Health checks: Implement proper health checks for all services
    4. Resource limits: Set appropriate resource limits
    5. Security: Follow Docker security best practices

    Production Checklist¶

    • Environment variables: All secrets configured securely
    • SSL certificates: Valid certificates installed and configured
    • Resource limits: CPU and memory limits set appropriately
    • Health checks: All services have health checks configured
    • Backups: Automated backup strategy implemented
    • Monitoring: Prometheus and Grafana configured
    • Logging: Centralized logging configured
    • Updates: Process for updating container images
    • Firewall: Network access properly restricted
    • Documentation: Deployment procedures documented
