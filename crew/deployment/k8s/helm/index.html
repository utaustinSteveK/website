    Kubernetes Deployment with Helm¶

    This guide covers deploying the Crew Appraisals System on Kubernetes using Helm charts, providing a production-ready, scalable deployment with high availability and automatic scaling capabilities.

    Overview¶

    Kubernetes deployment provides:

    • High Availability: Multi-replica deployments with load balancing
    • Auto-scaling: Horizontal Pod Autoscaler (HPA) for dynamic scaling
    • Service Discovery: Built-in DNS and service mesh capabilities
    • Rolling Updates: Zero-downtime deployments
    • Resource Management: CPU/memory limits and requests
    • Persistent Storage: StatefulSets for database persistence
    • Monitoring: Integrated with Prometheus and Grafana

    Prerequisites¶

    Cluster Requirements¶

    • Kubernetes: Version 1.24+ (tested with 1.28)
    • Helm: Version 3.8+
    • kubectl: Compatible with your cluster version
    • Storage: Dynamic volume provisioning
    • Ingress Controller: nginx-ingress or similar
    • Resources: Minimum 4 CPU cores, 8GB RAM

    Local Development Clusters¶

    Minikube¶

    # Install minikube
curl -Lo minikube https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
chmod +x minikube
sudo mv minikube /usr/local/bin/

# Start cluster
minikube start --cpus=4 --memory=8192 --disk-size=20g

# Enable addons
minikube addons enable ingress
minikube addons enable metrics-server

    Kind (Kubernetes in Docker)¶

    # Install kind
go install sigs.k8s.io/kind@v0.20.0

# Create cluster
kind create cluster --config=kind-config.yaml

# Install ingress controller
kubectl apply -f https://raw.githubusercontent.com/kubernetes/ingress-nginx/main/deploy/static/provider/kind/deploy.yaml

    # Install eksctl
curl --silent --location "https://github.com/weaveworks/eksctl/releases/latest/download/eksctl_$(uname -s)_amd64.tar.gz" | tar xz -C /tmp
sudo mv /tmp/eksctl /usr/local/bin

# Create cluster
eksctl create cluster --name crew-appraisals --region us-west-2 --nodegroup-name standard-workers --node-type m5.large --nodes 3

    apiVersion: v2

  - name: Platform Team
    email: platform@crew-appraisals.com

  - name: postgresql

  - fastapi

  - redis

  {{- if not .Values.autoscaling.enabled }}

  {{- end }}
  selector:
    matchLabels:
      {{- include "crew-appraisals.selectorLabels" . | nindent 6 }}
      app.kubernetes.io/component: backend
  template:
    metadata:
      annotations:
        checksum/config: {{ include (print $.Template.BasePath "/configmap.yaml") . | sha256sum }}
        checksum/secret: {{ include (print $.Template.BasePath "/secret.yaml") . | sha256sum }}
      labels:
        {{- include "crew-appraisals.selectorLabels" . | nindent 8 }}
        app.kubernetes.io/component: backend
    spec:
      {{- with .Values.security.podSecurityContext }}
      securityContext:
        {{- toYaml . | nindent 8 }}
      {{- end }}
      serviceAccountName: {{ include "crew-appraisals.serviceAccountName" . }}
      containers:
        - name: backend
          {{- with .Values.security.securityContext }}

  maxReplicas: {{ .Values.autoscaling.maxReplicas }}

{{- end }}

    Deployment Commands¶

    Install Helm and Dependencies¶

    # Install Helm
curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash

# Add repositories
helm repo add bitnami https://charts.bitnami.com/bitnami

helm repo update

# Install dependencies
cd helm/

    Deploy to Different Environments¶

    Development Deployment¶

    # Create namespace
kubectl create namespace crew-appraisals-dev

# Install/upgrade
helm upgrade --install crew-appraisals-dev ./helm \
  --namespace crew-appraisals-dev \
  --values helm/values-dev.yaml \
  --set app.image.tag="dev-$(git rev-parse --short HEAD)"

# Port forward for local access
kubectl port-forward -n crew-appraisals-dev service/crew-appraisals-dev 8080:80

    Staging Deployment¶

    # Create namespace
kubectl create namespace crew-appraisals-staging

# Install cert-manager for SSL
helm repo add jetstack https://charts.jetstack.io
helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --create-namespace \
  --version v1.13.0 \
  --set installCRDs=true

# Deploy application
helm upgrade --install crew-appraisals-staging ./helm \
  --namespace crew-appraisals-staging \
  --values helm/values-staging.yaml \
  --set config.app.secretKey="$(openssl rand -base64 32)"

    Production Deployment¶

    # Create namespace
kubectl create namespace crew-appraisals-prod

# Create secrets
kubectl create secret generic crew-appraisals-secrets \
  --namespace crew-appraisals-prod \
  --from-literal=secret-key="$(openssl rand -base64 32)" \
  --from-literal=db-password="$(openssl rand -base64 32)" \
  --from-literal=redis-password="$(openssl rand -base64 32)"

# Deploy with production values
helm upgrade --install crew-appraisals ./helm \
  --namespace crew-appraisals-prod \
  --values helm/values-production.yaml \
  --wait \
  --timeout=10m

# Verify deployment
kubectl get pods -n crew-appraisals-prod
kubectl get services -n crew-appraisals-prod
kubectl get ingress -n crew-appraisals-prod

    Monitoring and Maintenance¶

    # Check deployment status
helm status crew-appraisals -n crew-appraisals-prod

# View deployment history
helm history crew-appraisals -n crew-appraisals-prod

# Rollback to previous version
helm rollback crew-appraisals 1 -n crew-appraisals-prod

# Update deployment
helm upgrade crew-appraisals ./helm \
  --namespace crew-appraisals-prod \
  --values helm/values-production.yaml \
  --set app.image.tag="1.0.1"
