    Deployment Guide¶

    Platform Requirements

    Linux Only - NetIntel-OCR is tested and supported only on Linux systems.

    • Supported OS: Ubuntu 20.04/22.04, RHEL 8/9, Debian 11/12
    • Python: 3.11.x or 3.12.x ONLY
    • Architecture: x86_64 (ARM support planned)

    Windows and macOS deployments are not currently supported.

    Initialization¶

    NetIntel-OCR provides automated deployment configuration:

    # Generate deployment configurations
netintel-ocr --init

# Creates:
# - docker-compose.yml
# - kubernetes manifests
# - .env configuration

    Docker Deployment¶

    Single Container¶

    # Use Python 3.11 or 3.12 ONLY
FROM python:3.11-slim-bullseye
# OR: FROM python:3.12-slim-bookworm

WORKDIR /app

# Install system dependencies (Linux only)
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    python3-dev \
    && rm -rf /var/lib/apt/lists/*

# Install NetIntel-OCR from PyPI
# https://pypi.org/project/netintel-ocr/
RUN pip install netintel-ocr

# Configure Ollama endpoint
ENV OLLAMA_HOST=http://ollama:11434

CMD ["netintel-ocr", "--api"]

    Base Image Selection

    • Use python:3.11-slim-bullseye for Debian 11 base
    • Use python:3.12-slim-bookworm for Debian 12 base
    • Alpine Linux images are not supported due to dependency issues

    Docker Compose¶

    Generated docker-compose.yml:

    version: '3.8'

services:
  netintel-ocr:
    image: netintel-ocr:latest
    container_name: netintel-ocr
    environment:
      - OLLAMA_HOST=http://ollama:11434
      - API_PORT=8000
    ports:
      - "8000:8000"
    volumes:
      - ./input:/app/input
      - ./output:/app/output
      - ./config:/app/config
    depends_on:
      - ollama
    restart: unless-stopped

  ollama:
    image: ollama/ollama:latest
    container_name: ollama
    ports:
      - "11434:11434"
    volumes:
      - ollama_data:/root/.ollama
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]

  milvus:
    image: milvusdb/milvus:latest
    container_name: milvus
    ports:
      - "19530:19530"
      - "9091:9091"
    volumes:
      - milvus_data:/var/lib/milvus
    environment:
      - ETCD_ENDPOINTS=etcd:2379
      - MINIO_ADDRESS=minio:9000

volumes:
  ollama_data:
  milvus_data:

    Build and Run¶

    # Verify Linux system
uname -s  # Should output "Linux"

# Verify Python version in container
docker run --rm python:3.11-slim-bullseye python --version
# Output: Python 3.11.x

# Build image
docker build -t netintel-ocr:latest .

# Start services
docker-compose up -d

# Check logs
docker-compose logs -f netintel-ocr

    Platform Compatibility

    Docker images built on Linux may not work correctly on Windows/Mac Docker Desktop due to architecture differences.

    Kubernetes Deployment¶

    ConfigMap¶

    apiVersion: v1
kind: ConfigMap
metadata:
  name: netintel-ocr-config
  namespace: default
data:
  OLLAMA_HOST: "http://ollama-service:11434"
  API_PORT: "8000"
  LOG_LEVEL: "INFO"

    Deployment¶

    apiVersion: apps/v1
kind: Deployment
metadata:
  name: netintel-ocr
  namespace: default
spec:
  replicas: 2
  selector:
    matchLabels:
      app: netintel-ocr
  template:
    metadata:
      labels:
        app: netintel-ocr
    spec:
      containers:
      - name: netintel-ocr
        image: netintel-ocr:latest
        ports:
        - containerPort: 8000
        envFrom:
        - configMapRef:
            name: netintel-ocr-config
        volumeMounts:
        - name: input
          mountPath: /app/input
        - name: output
          mountPath: /app/output
        resources:
          requests:
            memory: "4Gi"
            cpu: "2"
          limits:
            memory: "8Gi"
            cpu: "4"
      volumes:
      - name: input
        persistentVolumeClaim:
          claimName: netintel-input-pvc
      - name: output
        persistentVolumeClaim:
          claimName: netintel-output-pvc

    Service¶

    apiVersion: v1
kind: Service
metadata:
  name: netintel-ocr-service
  namespace: default
spec:
  selector:
    app: netintel-ocr
  ports:
  - protocol: TCP
    port: 8000
    targetPort: 8000
  type: LoadBalancer

    Horizontal Pod Autoscaler¶

    apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: netintel-ocr-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: netintel-ocr
  minReplicas: 2
  maxReplicas: 10
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 70
  - type: Resource
    resource:
      name: memory
      target:
        type: Utilization
        averageUtilization: 80

    Deploy to Kubernetes¶

    # Apply configurations
kubectl apply -f k8s/

# Check deployment
kubectl get pods -l app=netintel-ocr

# View logs
kubectl logs -f deployment/netintel-ocr

# Scale deployment
kubectl scale deployment netintel-ocr --replicas=5

    Production Configuration¶

    System Requirements¶

    Production Environment

    • OS: Linux x86_64 (Ubuntu 20.04+ or RHEL 8+ recommended)
    • Python: 3.11.x or 3.12.x (verified installation)
    • Memory: 16GB minimum, 32GB recommended
    • CPU: 8+ cores recommended
    • Storage: SSD with 100GB+ available

    Environment Variables¶

    # Python version check (REQUIRED)
PYTHON_VERSION=$(python3 --version | cut -d' ' -f2)
# Must be 3.11.x or 3.12.x

# Required
OLLAMA_HOST=http://ollama.internal:11434
MILVUS_HOST=milvus.internal:19530

# Optional
API_PORT=8000
LOG_LEVEL=INFO
MAX_WORKERS=4
TIMEOUT_SECONDS=300
CACHE_DIR=/app/cache

    API Mode¶

    Start API Server¶

    # Local
netintel-ocr --api --port 8000

# Docker
docker run -p 8000:8000 netintel-ocr:latest --api

    API Endpoints¶

    # Process document
curl -X POST http://localhost:8000/process \
  -F "file=@document.pdf" \
  -F "model=qwen2.5vl:7b"

# Check status
curl http://localhost:8000/status/job-id

# Get results
curl http://localhost:8000/results/job-id
