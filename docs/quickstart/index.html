    Quick Start Guide¶

    Installation¶

    System Requirements¶

    Operating System Support

    Linux Only - NetIntel-OCR is currently tested and supported only on Linux distributions.

    Windows and macOS support is planned for future releases but not currently available.

    Python Version

    Python 3.11 or 3.12 Required - NetIntel-OCR is tested and supported only on Python 3.11 and 3.12.

    Other Python versions may work but are not officially supported.

    Verified Configurations¶

    • OS: Ubuntu 20.04/22.04, RHEL 8/9, Debian 11/12
    • Python: 3.11.x or 3.12.x
    • RAM: 8GB minimum (16GB recommended)
    • Storage: 10GB for models + processing space
    • Ollama: Version 0.1.0 or higher

    Python Setup¶

    # Check Python version (must be 3.11 or 3.12)
python3 --version

# Install Python 3.11 on Ubuntu/Debian
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-dev

# Or install Python 3.12
sudo apt install python3.12 python3.12-venv python3.12-dev

# Create virtual environment with Python 3.11
python3.11 -m venv venv
source venv/bin/activate

# Or with Python 3.12
python3.12 -m venv venv
source venv/bin/activate

    Install NetIntel-OCR¶

    # Ensure you're using Python 3.11 or 3.12
python --version  # Should show 3.11.x or 3.12.x

# Install latest version from PyPI
pip install netintel-ocr

# Or install specific version
pip install netintel-ocr==0.1.16.15

# Verify installation
netintel-ocr --version

    Package Information

    NetIntel-OCR is available on PyPI: https://pypi.org/project/netintel-ocr/

    Check for latest versions and release notes on the PyPI page.

    Install Ollama Models¶

    NetIntel-OCR requires vision-capable models for diagram processing:

    # Pull recommended models
ollama pull qwen2.5vl:7b        # For network diagrams
ollama pull Nanonets-OCR-s:latest  # For OCR processing

# Optional: Pull alternative models
ollama pull llava:13b            # Alternative vision model
ollama pull minicpm-v:latest     # Lightweight option

    Configuration¶

    Setting Ollama Host¶

    For remote Ollama servers:

    # Set environment variable
export OLLAMA_HOST="http://192.168.1.100:11434"

# Or use config file
netintel-ocr --init

    Model Selection¶

    Specify models for different processing tasks:

    Parameter Purpose Recommended Model
    --model OCR and text extraction Nanonets-OCR-s:latest
    --network-model Network diagram analysis qwen2.5vl:7b
    --flow-model Flow chart processing qwen2.5vl:7b

    Basic Usage¶

    Process a PDF Document¶

    # Process entire document
netintel-ocr document.pdf

# Process specific pages
netintel-ocr --start 5 --end 10 document.pdf

# Enable debug output
netintel-ocr --debug document.pdf

    Example: Network Architecture Document¶

    netintel-ocr \
  --model Nanonets-OCR-s:latest \
  --network-model qwen2.5vl:7b \
  --debug \
  cisco-sdwan-design-guide.pdf \
  --start 5 --end 10

    Output structure: 

    output/
├── cisco-sdwan-design-guide/
│   ├── page_005.md          # Extracted text
│   ├── page_006_network.md  # Network diagram with Mermaid
│   ├── page_007.md          # Regular page
│   └── summary.json         # Processing summary

    Output Formats¶

    Markdown Files¶

    Each page generates a markdown file containing: - Extracted text content - Mermaid diagrams for network/flow charts - Context analysis and interpretations - Component and connection listings

    Mermaid Diagrams¶

    Network diagrams are converted to Mermaid syntax:

    graph TB
    Router["Core Router"]
    Switch1["Access Switch 1"]
    Switch2["Access Switch 2"]

    Router --> Switch1
    Router --> Switch2

    Summary JSON¶

    Processing statistics and metadata:

    {
  "total_pages": 10,
  "network_diagrams": 3,
  "flow_diagrams": 2,
  "processing_time": 45.2,
  "models_used": {
    "ocr": "Nanonets-OCR-s:latest",
    "network": "qwen2.5vl:7b"
  }
}

    Common Use Cases¶

    Extract Network Topologies¶

    netintel-ocr \
  --network-only \
  --network-model qwen2.5vl:7b \
  network-design.pdf

    Process Security Documentation¶

    netintel-ocr \
  --include-context \
  --extract-security \
  firewall-config.pdf

    Batch Processing¶

    # Process multiple documents
for pdf in *.pdf; do
  netintel-ocr --output-dir processed/ "$pdf"
done

    Troubleshooting¶

    Model Not Found¶

    # Check available models
ollama list

# Pull missing model
ollama pull qwen2.5vl:7b

    Connection Issues¶

    # Test Ollama connection
curl http://localhost:11434/api/tags

# Check environment
echo $OLLAMA_HOST

    Performance Optimization¶

    # Use faster model for testing
netintel-ocr --network-model minicpm-v:latest document.pdf

# Limit pages for large documents
netintel-ocr --start 1 --end 20 large-document.pdf

# Disable context extraction for speed
netintel-ocr --no-context document.pdf
