    Knowledge Graph Model Configuration¶

    NetIntel-OCR v0.1.17 supports configurable OLLAMA models for embedding and inference in the Knowledge Graph CLI.

    Model Configuration Methods¶

    1. Command-Line Options¶

    Most KG commands that use MiniRAG or HybridRetriever now support model selection:

    # Specify models directly
netintel-ocr kg rag-query "Find compliance issues" \
  --llm-model "ollama/gemma3:4b-it-qat" \
  --embedding-model "ollama/Qwen3-Embedding-8B"

# Other commands with model support
netintel-ocr kg entity-context ENTITY_ID \
  --llm-model "ollama/llama3:8b" \
  --embedding-model "ollama/nomic-embed-text"

netintel-ocr kg hybrid-search "security vulnerabilities" \
  --llm-model "ollama/mistral:7b" \
  --embedding-model "ollama/mxbai-embed-large"

    2. Environment Variables¶

    Set default models using environment variables:

    # Set LLM model for inference
export MINIRAG_LLM="ollama/gemma3:4b-it-qat"

# Set embedding model
export MINIRAG_EMBEDDING="ollama/Qwen3-Embedding-8B"

# Set embedding dimensions (must match model)
export MINIRAG_EMBEDDING_DIM="4096"

# Configure OLLAMA server
export OLLAMA_HOST="localhost"
export OLLAMA_PORT="11434"

    3. Supported Commands¶

    The following commands support --llm-model and --embedding-model options:

    • rag-query: Query using Enhanced MiniRAG with KG embeddings
    • entity-context: Get rich context for an entity using MiniRAG
    • hybrid-search: Perform hybrid search with advanced retrieval strategies
    • compare-strategies: Compare different retrieval strategies
    • batch-query: Process batch queries from file

    Available OLLAMA Models¶

    Recommended LLM Models¶

    • ollama/gemma3:4b-it-qat (Default, optimized for speed)
    • ollama/llama3:8b (Better quality, slower)
    • ollama/mistral:7b (Good balance)
    • ollama/qwen2.5:7b (Multilingual support)

    Recommended Embedding Models¶

    • ollama/Qwen3-Embedding-8B (Default, 4096 dimensions)
    • ollama/nomic-embed-text (768 dimensions, faster)
    • ollama/mxbai-embed-large (1024 dimensions)
    • ollama/bge-large (1024 dimensions)

    Examples¶

    Basic Query with Custom Models¶

    netintel-ocr kg rag-query "What are the latest security patches?" \
  --llm-model "ollama/llama3:8b" \
  --embedding-model "ollama/nomic-embed-text" \
  --mode hybrid

    Entity Context with Environment Variables¶

    # Set defaults
export MINIRAG_LLM="ollama/mistral:7b"
export MINIRAG_EMBEDDING="ollama/mxbai-embed-large"

# Use defaults
netintel-ocr kg entity-context USER_123 --context-size 3

    Batch Processing with Custom Models¶

    netintel-ocr kg batch-query queries.txt \
  --llm-model "ollama/qwen2.5:7b" \
  --embedding-model "ollama/bge-large" \
  --output results.json

    Strategy Comparison¶

    netintel-ocr kg compare-strategies "compliance violations" \
  --llm-model "ollama/gemma3:4b-it-qat" \
  --embedding-model "ollama/Qwen3-Embedding-8B" \
  --limit 10

    Model Selection Guidelines¶

    For Speed¶

    • LLM: ollama/gemma3:4b-it-qat
    • Embedding: ollama/nomic-embed-text

    For Quality¶

    • LLM: ollama/llama3:8b or ollama/mistral:7b
    • Embedding: ollama/Qwen3-Embedding-8B

    For Multilingual¶

    • LLM: ollama/qwen2.5:7b
    • Embedding: ollama/multilingual-e5-large

    Checking Available Models¶

    To see what models are available on your OLLAMA server:

    # List all available models
ollama list

# Check if specific model is available
ollama show gemma3:4b-it-qat

    Troubleshooting¶

    Model Not Found¶

    If you get a "model not found" error, pull the model first: 

    ollama pull gemma3:4b-it-qat
ollama pull Qwen3-Embedding-8B

    Dimension Mismatch¶

    Ensure MINIRAG_EMBEDDING_DIM matches your embedding model: - Qwen3-Embedding-8B: 4096 - nomic-embed-text: 768 - mxbai-embed-large: 1024

    Connection Issues¶

    Check OLLAMA server: 

    netintel-ocr kg check-requirements --verbose

    This will show if OLLAMA is accessible and list available models.
